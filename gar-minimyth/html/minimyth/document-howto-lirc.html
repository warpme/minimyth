LIRC: Using a remote control
Introduction

MiniMyth supports remote controls using a combination of udev, LIRC, eventlircd, and MiniMyth read only configuration variables. This combination enables MiniMyth

  • to support remote control hotplugging,
  • to support remote controls that may not be present when MiniMyth boots (e.g. Sony Play Station 3 Blu-ray Disc Remote Control),
  • to support remote controls that appear as Linux event devices and use keyboard shortcuts (e.g. Adesso ARC-1100 Remote Control),
  • to support remote controls that contain mouse/joystick controls (e.g. Adesso ARC-1100 Remote Control), and
  • to support multiple remote controls.

MiniMyth uses udev

  • to load LIRC kernel driver(s) when a device requiring the driver(s) is added,
  • to start and stop lircd when a device requiring lircd is added or removed, and
  • to notify eventlircd when an input event device that should be routed through LIRC is added or removed.

MiniMyth uses the LIRC kernel drivers (lirc_*.ko) and the LIRC daemon (lircd)

  • to create a Linux input event device for receiving remote control key presses as Linux input events, and
  • to create an lircd socket for sending IR commands using IR transmitters.

MiniMyth uses the eventlircd daemon

  • to map keyboard shortcuts to single events,
  • to create a Linux input event device for events not handled by the lircd socket (e.g. mouse and joystick events), and
  • to combine multiple input event devices into a single lircd socket.

If your MiniMyth system has a remote control receiver/transmiiter that is not (or cannot) be detected by udev, then you can manually configure it using the MM_LIRC_* configuration variables. When possible, MiniMyth uses the MM_LIRC_* configuration variables to modify the udev scripts so that the remote control is handled by the normal udev->lircd->eventlircd path. Otherwise, MiniMyth uses an init script to start the appropriate daemon for the manually configured remote control.
